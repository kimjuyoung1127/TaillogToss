## 1. 포지셔닝

> **TailLog는 반려견 유치원·호텔의 매출을 다루는 서비스가 아니라,**
> **강아지 상태 관리와 보호자 커뮤니케이션을 고도화하는 B2B 관리 SaaS이다.**
> **매장 이용요금 결제는 각 매장에서 별도로 처리한다.**

**다루는 것**: 출석/컨디션/놀이/이상징후, 보호자 리포트, 기록완료율/열람률
**절대 안 다루는 것**: 이용요금, 보호자 결제내역, 정산, 매출분석 → POS 오해 방지 → QA 심사 단순화

---

## 2. 3가지 사용자 모드

| 모드 | 대상 | 핵심 화면 |
|------|------|----------|
| **일반 사용자** | B2C 보호자 (기존) | 대시보드, 행동기록, AI코칭, 훈련 |
| **종사자 (센터)** | 유치원/호텔 직원 | Today Ops Queue, Bulk기록, 리포트 |
| **종사자 (개인 훈련사)** | 프리랜서 훈련사 | 내 담당 큐, 리포트, 보호자 소통 |

> 종사자 모드는 **조직(센터) 소속 종사자** + **개인 훈련사** 모두 사용 가능.
> 개인 훈련사는 `dog_assignments` 기반으로 맡은 강아지만 접근.

---

## 3. 확정 사항

- **결제**: 토스 IAP (디지털 서비스 구독). 센터 플랜 + 개인 훈련사 플랜 2레일
- **대시보드**: 토스 미니앱 올인 (별도 웹앱 없음)
- **스키마**: v1에 B2B 테이블 10개 빈 상태 포함 (B2C 무영향)
- **dogs.user_id**: NOT NULL 유지 (B2B는 `org_dogs`로 관리, dogs 테이블 안 건드림)

## 4. 운영 UX (Today Ops Queue)

### 4.1 핵심 원칙

> **40마리 운영은 "강아지 목록"이 아니라 "Today Ops Queue" 1화면에서 80% 처리.**
> **키보드 입력 최소화 — 프리셋 원탭 80%, 예외만 텍스트 20%.**

### 4.2 화면 구조 (진짜 2페이지)

```
pages/
├── ops/today             ★ 운영 원스탑 (대부분의 업무 여기서)
│   ├── [탭] 미기록         업무 상태 필터 기본
│   ├── [탭] 주의 필요       이상징후/급증
│   ├── [탭] 리포트 미발송    오늘 리포트 아직 안 간 건
│   ├── [탭] 내 담당         ★ 개인 훈련사/담당자 전용
│   ├── [바텀시트] Bulk Edit  다중 선택 → 프리셋 일괄 기록
│   ├── [바텀시트] 리포트 미리보기/편집/발송
│   ├── [바텀시트] 보호자 질문 빠른 답변
│   └── [바텀시트] 통계 요약 (기록완료율, 열람률) — 드로어 대신 바텀시트 (토스 UX 관습)
├── ops/settings          조직/직원/템플릿/발송시간/프리셋 커스텀
├── parent/reports        보호자: 리포트 열람 + 리액션
└── report/[shareToken]   비토스 보호자용 공개 링크
```

> **Note**: Granite 파일 기반 라우팅 (`pages/` → `src/pages/` 2단계 구조). 40마리 카드는 FlatList 네이티브 가상화로 60fps 보장.

나머지는 전부 `ops/today`의 **탭/바텀시트**로 흡수. (드로어 대신 바텀시트 — 토스 UX 관습)

### 4.3 Bulk 프리셋 스펙

**원칙**: 키보드 입력 0에 가깝게. 80%는 원탭, 20%만 텍스트.

| 카테고리 | 프리셋 옵션 | UI |
|----------|-----------|-----|
| **산책** | 10분 / 20분 / 30분 | 칩 선택 |
| **놀이** | 그룹 A / B / C / 개별 | 칩 선택 |
| **컨디션** | 좋음 / 보통 / 주의 | 3색 버튼 |
| **이상징후** | 설사 / 식욕저하 / 격리 / 과흥분 / 기타 | 칩 (기타만 텍스트) |
| **식사** | 완식 / 반식 / 거부 | 3색 버튼 |
| **사회성** | 좋음 / 보통 / 주의 (특정 견과 충돌) | 칩 + 선택적 메모 |

**Bulk 플로우**: FlatList 카드 다중 선택 → 상단 액션바 → 프리셋 선택 → 일괄 저장 (Reanimated 피드백)

### 4.4 종사자 모드 UX

**종사자 모드 진입 시 2탭**:
- **[내 담당]**: `dog_assignments` 기반. 센터 직원이든 개인 훈련사든 동일 화면
- **[전체/센터]**: 센터 소속이면 센터 전체 (권한 따라). 개인 훈련사는 이 탭 없음

**"내 담당" 화면이 킬러**:
- "오늘 해야 할 기록" 큐 (미기록 강아지)
- "미응답 질문" 큐 (보호자 질문)
- "리포트 발송 예정" 큐 (생성완료 + 미발송)

## 6. 결제 & 과금

### 6.1 IAP 상품 구조

| 플랜 | 대상 | 가격(월) | max_dogs | max_staff |
|------|------|---------|----------|-----------|
| **센터 Basic** | 유치원/호텔 | 29,000원 | 30 | 5 |
| **센터 Pro** | 대형 유치원 | 59,000원 | 60 | 10 |
| **센터 Enterprise** | 체인 | 99,000원 | 100+ | 20 |
| **훈련사 Lite** | 개인 훈련사 | 9,900원 | 10 | 1 |
| **훈련사 Standard** | 중견 훈련사 | 19,900원 | 30 | 1 |
| **훈련사 Pro** | 전문 훈련사 | 39,900원 | 50 | 1 |

**핵심 규칙**:
- 센터 소속 훈련사 → 센터 플랜 안에서 사용 (개인 플랜 불필요)
- 프리랜서 훈련사 → 개인 플랜 결제해야 종사자 모드 활성화
- IAP 수수료(앱마켓 15–30% + 토스 5%)를 요금/원가 모델에 반영

### 6.2 Entitlement 서버 흐름

**절대 규칙**: IAP는 "결제 UI"만 해결. 권한 강제는 서버 책임.

```
결제 성공 → verify-iap-order (Edge Function) → 영수증 확인
  → org_subscriptions 갱신 (status=active, max_dogs, max_staff, expires_at)
  → Entitlement 강제:
    - org_dogs 등록 시: COUNT(*) < max_dogs 검증
    - org_members 초대 시: COUNT(*) < max_staff 검증
    - daily_reports 생성 시: subscription.status = 'active' 검증
    - 만료 시: 리포트 생성 차단 (기존 데이터 열람은 유지)
```

**만료 처리**:
- `expires_at` 초과 시 `status='expired'` 자동 전환 (CRON)
- 즉시 차단: 신규 기록/리포트 생성 불가
- 유예: 기존 데이터 열람/내보내기는 30일 유지

**구독 상태 전이**:
```
trial → active (결제 성공)
active → expired (만료 CRON)
active → cancelled (사용자 취소 — 잔여기간 유지)
active → suspended (결제 실패 재시도 중)
suspended → active (결제 재성공)
suspended → expired (재시도 횟수 초과)
* → refunded (IAP 환불 웹훅 수신)
```

## 7. Phase 시퀀싱

```
Phase 0–6 (9–10주):  Toss B2C v1 론칭 (B2B 스키마 10개 테이블 빈 상태 포함)
  ↓ (안정화 2주)
Phase 7  (2주):      B2B RLS 벤치마크 + AI 리포트 프로토타입 + Entitlement 설계
Phase 8  (3–4주):    Layer 1 — Today Ops Queue + Bulk 프리셋 + dog_assignments
Phase 9  (3–4주):    Layer 2 — Auto Report Engine + 보호자 공유 (share_token + getTossShareLink)
Phase 10 (2–3주):    Layer 3 — Interactive Parent + 종사자 "내 담당" 탭
Phase 11 (2–3주):    Layer 4 — 운영 통계 + 개인 훈련사 플랜
Phase 12 (2주):      B2B QA + Toss 재심사
총: v1 이후 약 14–18주
```

**v1에 미리 포함**: 10개 신규 테이블 DDL (빈 상태) + users.role 확장 + behavior_logs.org_id nullable + RLS 헬퍼 함수

## 9. 네트워크 효과

```
보호자: "센터 + 내 담당 훈련사"가 같은 앱 → 신뢰 ↑, 이탈 ↓
훈련사: 숙제/기록/리포트 자동화 → 업무 효율, 고객 유지
TailLog: 센터 없이 개인 훈련사부터 확장 가능 → 훈련사→센터 역확산
```
